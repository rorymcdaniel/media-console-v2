In src/app/AppBuilder.cpp:
- Add include: `#include "cd/CdController.h"`
- In build(), after creating m_localPlaybackController, add:

```cpp
// Create CD subsystem controller
m_cdController = std::make_unique<CdController>(
    m_cdDrive.get(),
    m_localPlaybackController.get(),
    m_playbackState.get(),
    config.cd,
    this
);

// Start CD disc detection polling
m_cdController->start();

qCInfo(mediaApp) << "AppBuilder: CD controller initialized";
```

- Add to context building:
```cpp
ctx.cdController = m_cdController.get();
```

This follows the exact same pattern used for ReceiverController and LocalPlaybackController: AppBuilder owns via unique_ptr, AppContext holds non-owning pointer. CdController starts automatically on build, just like ReceiverController (which calls start() in build()).

5. **CD subsystem (cdController)** <-- NEW, after localPlaybackController
