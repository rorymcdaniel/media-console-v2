In src/app/AppBuilder.cpp:
- Add conditional include:
```cpp

#include "library/FlacLibraryController.h"
#endif
```
- In build(), after creating m_cdController and its start() call, add:
```cpp
// Create FLAC library controller (Linux only with libsndfile/taglib)
#ifdef HAS_SNDFILE
m_flacLibraryController = std::make_unique<FlacLibraryController>(
    m_localPlaybackController.get(),
    m_playbackState.get(),
    config.library,
    this
);
m_flacLibraryController->start();
qCInfo(mediaApp) << "AppBuilder: FLAC library controller initialized";
#else
qCInfo(mediaApp) << "AppBuilder: FLAC library disabled (no libsndfile)";
#endif
```

- Add to context building:
```cpp
#ifdef HAS_SNDFILE
ctx.flacLibraryController = m_flacLibraryController.get();
#endif
```

This follows the exact same pattern used for CdController: AppBuilder owns via unique_ptr, AppContext holds non-owning pointer. FlacLibraryController starts automatically on build. On macOS (no HAS_SNDFILE), the controller is not created and the context pointer remains nullptr.
