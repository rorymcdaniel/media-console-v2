<objective>
Implement the Spotify OAuth 2.0 PKCE authentication layer with token persistence and CLI-based authorization flow.

Purpose: Authentication is the foundation for all Spotify API access. Without valid tokens, no search, playback, or device control is possible. The CLI auth mode provides a kiosk-friendly way to authorize without a touchscreen login UI (locked user decision).

Output: SpotifyAuth class managing OAuth lifecycle, CMake NetworkAuth integration, CLI auth entry point in main.cpp.
</objective>

       ```cpp
       #pragma once

       #include <QObject>
       #include <QOAuth2AuthorizationCodeFlow>
       #include <QOAuthHttpServerReplyHandler>

       struct SpotifyConfig;

       class SpotifyAuth : public QObject
       {
           Q_OBJECT

       public:
           explicit SpotifyAuth(const SpotifyConfig& config, QObject* parent = nullptr);

           bool isAuthenticated() const;
           QString accessToken() const;
           void startAuthFlow();
           bool restoreTokens();
           void saveTokens();
           void clearTokens();

       signals:
           void authStateChanged(bool authenticated);
           void authorizationUrlReady(const QUrl& url);
           void authFlowComplete();
           void authError(const QString& error);

       private slots:
           void onGranted();
           void onRefreshTokenChanged(const QString& refreshToken);

       private:
           void setupOAuth(const SpotifyConfig& config);

           QOAuth2AuthorizationCodeFlow m_oauth;
           QOAuthHttpServerReplyHandler* m_replyHandler = nullptr;
           bool m_authenticated = false;
       };
       ```

    4. Create src/spotify/SpotifyAuth.cpp:
       - Constructor: call setupOAuth(config)
       - setupOAuth(): Configure m_oauth with:
         - Authorization URL: https://accounts.spotify.com/authorize
         - Token URL: https://accounts.spotify.com/api/token
         - Client ID from config.clientId
         - setPkceMethod(QOAuth2AuthorizationCodeFlow::PkceMethod::S256)
         - Scopes: user-read-playback-state, user-modify-playback-state, user-read-currently-playing, playlist-read-private, playlist-read-collaborative
         - setAutoRefresh(true), setRefreshLeadTime(std::chrono::seconds(300))
         - Create QOAuthHttpServerReplyHandler on config.redirectPort
         - setCallbackText("Authorization complete. You can close this tab and return to your terminal.")
         - Connect QAbstractOAuth::authorizeWithBrowser to authorizationUrlReady
         - Connect QAbstractOAuth::granted to onGranted
         - Connect QAbstractOAuth2::refreshTokenChanged to onRefreshTokenChanged
       - isAuthenticated(): return m_authenticated and token is not empty
       - accessToken(): return m_oauth.token()
       - startAuthFlow(): start reply handler listening, call m_oauth.grant()
       - restoreTokens(): read from QSettings [spotify_auth] group (access_token, refresh_token, expiry). If tokens found, set on m_oauth, set m_authenticated=true, return true. If expired but refresh token exists, call m_oauth.refreshTokens(). Return false if no tokens.
       - saveTokens(): write access_token, refresh_token, expiry to QSettings [spotify_auth] group using m_oauth.token(), m_oauth.refreshToken(), m_oauth.expirationAt()
       - clearTokens(): remove all keys from QSettings [spotify_auth], set m_authenticated=false, emit authStateChanged(false)
       - onGranted(): set m_authenticated=true, saveTokens(), emit authStateChanged(true), emit authFlowComplete(), close reply handler
       - onRefreshTokenChanged(): saveTokens() to handle Spotify token rotation
