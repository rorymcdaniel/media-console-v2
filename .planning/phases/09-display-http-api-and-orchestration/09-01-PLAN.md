    - autoDetectDisplay() returns false and logs warning when ddcutil is not installed or no display found
    - setBrightness(percent) runs `ddcutil setvcp 10 {percent} --bus {busNumber}`, returns true on exit code 0, emits brightnessChanged

    - brightness() returns cached m_brightness value; isPowered() returns cached m_powered value

    - Constructor takes QObject* parent; stores bus number as m_busNumber (default -1 = not detected)

  <behavior>
    - Initial state is ACTIVE with full brightness
    - After dimTimeoutMs of inactivity, transitions to DIMMED (setBrightness(dimBrightness))
    - After offTimeoutMs of inactivity, transitions to OFF (setPower(false))
    - activityDetected() from any state: setPower(true), setBrightness(100), restart dim timer, cancel off timer
    - When PlaybackState::playbackMode == Playing: suppress all timers (screen stays on)
    - When playbackMode changes from Playing to Stopped/Paused: restart timers from current state
    - doorOpen (UIState::doorOpenChanged(true)): activityDetected()
    - doorClose (UIState::doorOpenChanged(false)): immediate transition to DIMMED then OFF (skip dim timer)
    - When timeoutEnabled is false in config: no timers run, stays ACTIVE
    - Smooth dimming from ACTIVE to DIMMED: step brightness down over ~1 second using QTimer steps
    - UIState gets new Q_PROPERTY screenDimmed (bool) set by ScreenTimeoutController
  </behavior>
  <action>
    **ScreenTimeoutController.h:**
    ```cpp
    enum class ScreenState { Active, Dimming, Dimmed, Off };
    ```
    - Constructor takes IDisplayControl*, PlaybackState*, UIState*, const DisplayConfig&, QObject* parent
    - Public slots: activityDetected(), start(), stop()
    - Private slots: onDimTimeout(), onOffTimeout(), onDimStep(), onPlaybackModeChanged(PlaybackMode), onDoorOpenChanged(bool)
    - Private: ScreenState m_state, QTimer m_dimTimer, QTimer m_offTimer, QTimer m_dimStepTimer, int m_targetBrightness, int m_currentStepBrightness
    - Signal: stateChanged(ScreenState)
    - Q_PROPERTY screenState (exposed for debugging/logging)

    **ScreenTimeoutController.cpp:**
    - Constructor: connect PlaybackState::playbackModeChanged, UIState::doorOpenChanged, store config values
    - start(): if timeoutEnabled, set state ACTIVE, start dim timer (dimTimeoutMs)
    - activityDetected(): if state != Active, restore brightness to 100 and power on. Always restart dim timer. Stop off timer.
    - onDimTimeout(): Begin dimming animation. Set state to Dimming. Start dim step timer (50ms interval). Target brightness = dimBrightness.
    - onDimStep(): Decrease brightness by step (calculate step from (100 - dimBrightness) / 20 = ~4% per step for ~1s total). When reached target, set state DIMMED, start off timer (offTimeoutMs - dimTimeoutMs).
    - onOffTimeout(): setPower(false), set state Off, set UIState::screenDimmed(true)
    - onPlaybackModeChanged(): if Playing, stop all timers (suppress). If not Playing and state was Active due to playback, restart dim timer.
    - onDoorOpenChanged(bool open): if open, activityDetected(). If close, immediately start dim step (bypass dim timer) then schedule off quickly (e.g., 2 second delay).

    **UIState additions:**
    - Add Q_PROPERTY bool screenDimmed with getter/setter/signal (for Phase 10 QML binding)
    - Default false

    **tests/test_ScreenTimeoutController.cpp:**

    - Test activityDetected() resets to ACTIVE with brightness 100
    - Test playback suppression: playing = no timeout transition
    - Test door close triggers immediate dimming
    - Test door open triggers activityDetected
    - Use QTest::qWait() or QSignalSpy with short timer values (set dim timeout to 100ms for tests)
