  <behavior>
    - HttpApiServer(ReceiverController*, ReceiverState*, PlaybackState*, IDisplayControl*, SpotifyAuth*, SpotifyController*, const ApiConfig&, QObject* parent)
    - start() creates QTcpServer, listens on configured port, binds QHttpServer to it, returns bool success
    - stop() closes TCP server
    - POST /api/volume: parse JSON body {"volume": int}, call ReceiverController::setVolume, return {"status": "ok"}
    - POST /api/input: parse JSON body {"input": string}, map to MediaSource enum, call ReceiverController::selectInput, return {"status": "ok"}
    - POST /api/display: parse JSON body {"power": bool}, call IDisplayControl::setPower, return {"status": "ok"}
    - GET /api/status: return JSON with volume, powered, muted, input, displayPowered, displayBrightness, playbackMode, activeSource
    - GET /setup/spotify: call SpotifyAuth::startAuthFlow(), return HTML page with green Connect button linking to OAuth URL
    - GET /auth/spotify/callback: parse code/error from query params, display success or error HTML (actual token exchange is handled by SpotifyAuth's QOAuthHttpServerReplyHandler on port 8888)
    - GET /spotify/status: return JSON with authenticated status
    - POST /spotify/play: call SpotifyController::play() if authenticated, return {"status": "ok"} or {"error": "Not authenticated"}
    - All POST endpoints emit activityDetected signal (for ScreenTimeoutController to connect to in Plan 03)
  </behavior>
  <action>
    Create src/api/ directory. Implement HttpApiServer following the reference implementation pattern from ~/Code/media-console/src/api/HttpApiServer.cpp.

    **HttpApiServer.h:**
    ```cpp
    class HttpApiServer : public QObject {
        Q_OBJECT
    public:
        HttpApiServer(ReceiverController* receiverController,
                      ReceiverState* receiverState,
                      PlaybackState* playbackState,
                      IDisplayControl* displayControl,
                      SpotifyAuth* spotifyAuth,
                      SpotifyController* spotifyController,
                      const ApiConfig& config,
                      QObject* parent = nullptr);
        bool start();
        void stop();
        QString serverUrl() const;

    signals:
        void activityDetected();  // Emitted on every API request — connect to ScreenTimeoutController

    private:
        void setupRoutes();
        #ifdef QT_SSL_AVAILABLE
        bool setupSSL();
        bool generateSelfSignedCertificate();
        QString getCertificatePath() const;
        QString getKeyPath() const;
        #endif

        QHttpServer* m_server;
        QTcpServer* m_tcpServer;
        ReceiverController* m_receiverController;
        ReceiverState* m_receiverState;
        PlaybackState* m_playbackState;
        IDisplayControl* m_displayControl;
        SpotifyAuth* m_spotifyAuth;
        SpotifyController* m_spotifyController;
        int m_port;
        // SSL members under #ifdef QT_SSL_AVAILABLE
    };
    ```

    **HttpApiServer.cpp:**
    - Constructor: create QHttpServer, create QTcpServer (or QSslServer if SSL available), store all non-owning pointers, call setupRoutes()
    - start(): Attempt SSL setup (if available), listen on m_port, bind QHttpServer to QTcpServer. Follow reference pattern exactly.
    - setupRoutes(): Register all routes using m_server->route() lambdas. Each POST route calls emit activityDetected() before processing.
    - Input string mapping: "streaming"/"net" -> MediaSource::Streaming, "phono" -> Phono, "cd" -> CD, "computer" -> Computer, "bluetooth" -> Bluetooth, "library" -> Library. Use a static QMap or switch.
    - Spotify OAuth HTML pages: Lift directly from reference implementation (green Connect button, success/error pages)
    - SSL setup: Identical to reference — openssl subprocess for self-signed cert, stored in AppConfigLocation

    **CMakeLists.txt updates:**
    - Add `find_package(Qt6 REQUIRED COMPONENTS HttpServer)` to the Qt6 find_package line
    - Add HttpApiServer.h/.cpp to LIB_SOURCES
    - Add `Qt6::HttpServer` to target_link_libraries for media-console-lib
    - Note: The reference uses `#ifdef QT_SSL_AVAILABLE` — this is automatically defined when Qt6 is built with SSL support. No explicit find needed.

    **Error response format (Claude's discretion):**
    All error responses return JSON: {"error": "description"}. HTTP status codes: 200 for success, 400 for bad request (missing/invalid params), 404 for unknown route (QHttpServer default).
