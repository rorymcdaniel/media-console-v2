    ```cpp
    QVariantList CdController::tocAsVariantList() const
    {
        QVariantList result;
        const QVector<CdTrackInfo>& trackInfo = m_currentMetadata.tracks;

        for (const TocEntry& entry : m_currentToc) {
            QVariantMap item;
            item[QStringLiteral("trackNumber")] = entry.trackNumber;
            item[QStringLiteral("durationSeconds")] = entry.durationSeconds;

            // Look up title from metadata if available (tracks are 1-indexed like TOC)
            QString title;
            for (const CdTrackInfo& info : trackInfo) {
                if (info.trackNumber == entry.trackNumber) {
                    title = info.title;
                    break;
                }
            }
            item[QStringLiteral("title")] = title;  // empty string if not yet loaded

            result.append(item);
        }
        return result;
    }
    ```

    Add required includes to CdController.cpp if not already present:
    - `#include "cd/CdMetadataCache.h"` (for CdTrackInfo — already included transitively via CdController.h, verify)
    - `#include <QVariant>` (for QVariantList/QVariantMap)

    The NOTIFY signal for the Q_PROPERTY is the existing `tocReady` signal — no new signal needed.
    When QML binds to `CdController.toc`, it will re-evaluate when `tocReady` fires.

    Additionally, emit `tocReady` again (or an equivalent) when `metadataReady` fires, so QML
    track list updates its titles when metadata loads. In `onMetadataReady()` slot, after storing
    `m_currentMetadata`, emit `tocReady(m_currentToc)` to trigger QML re-binding:
    Find `onMetadataReady` in CdController.cpp and add `emit tocReady(m_currentToc);` after the
    metadata assignment so titles appear as soon as metadata loads.
